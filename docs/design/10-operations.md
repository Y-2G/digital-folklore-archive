# 運用ルール・実装ロードマップ

## 運用ルール

### 収蔵ID

- `DTA-000001` 形式で固定
- URL永続（ID変更しない）

### 創作の扱い

- Type を `ORIGINAL` として明示
- 既存伝承と混同しない（資料情報にも明記）

### 確度の更新

- `UNKNOWN` → `SECONDARY` / `PRIMARY` へ上げるのはOK
- 逆方向は慎重（版履歴に残す）

### タグの抑制

- motifs 最大3（推奨）
- タグ増殖は避け、必要なら「辞書」を先に更新する

## 実装ロードマップ

### Phase 1: 最小実装

1. **Catalog（一覧）→ Item（詳細）表示の最小実装**
   - Firestoreからのデータ取得
   - 基本的なUI表示

### Phase 2: 管理機能

2. **管理者用の追加・編集フロー**
   - まずは手動投入でもOK
   - 管理画面は後回し可

### Phase 3: 検索強化

3. **searchTokens 生成の自動化**
   - 管理画面 or Cloud Functions
   - 既存データへのバックフィル

### Phase 4: コレクション

4. **Collections（棚）追加**
   - 編集者キュレーション機能
   - 棚一覧・詳細ページ

### Phase 5: 信頼性向上

5. **Analysis / About 整備**
   - 統計・分布表示
   - 運営方針の明文化
   - 信頼性を上げる
