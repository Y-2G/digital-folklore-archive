## 概要

### 実装計画での重大観点
設計・タスク分解・リリースまでの段階で確認すべき7つの観点を定義しています。
- 目的と不変条件の言語化
- 仕様の境界（Edge case）
- 失敗時の挙動設計
- データ設計と移行の安全性
- リリース戦略
- 性能・コストの見積もり
- テスト戦略

### コードレビューでの重大観点
事故を防ぐためのコードレビュー時のチェック項目を7つの観点で整理しています。
- 正しさ（Correctness）
- 安全性（Security / Privacy）
- データ整合性（Integrity）
- 冪等性・再試行耐性（Resilience）
- 性能（Performance）
- 可観測性・運用（Observability）
- 変更容易性（Maintainability）
